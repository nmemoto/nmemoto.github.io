# ヘッダー 1

`aws cdk`を有用と感じているので、多用していきたいと思っている。そう感じている理由について書いてみる。

- 典型的な構成は`Construct`で簡単に作成可能。
- 一度作った構成は何度でも作れる。
- AWS リソースやそのまとまりの依存関係の定義が簡単。
  - Stack 間
    - ネットワーク・データストア・アプリケーション用の Stack 等、役割や変更度合いに応じた Stack の分割
    - Stack 間の入力が簡単
  - AWS リソース間
    - 関連するリソースを紐付けるだけ
- 黙っていても改善される
